# Naming Conventions

AIUI names should be stable, semantic, and easy for AI agents to understand.

## General Rules

- Use lowercase
- Use hyphen-separated names
- Prefer semantic names over visual names
- Avoid framework-specific terms
- Avoid brand imitation names

## Good Names

```text
management-list
settings-section
filter-toolbar
data-table-workspace
detail-drawer
precise-product-ui
calm-saas
editorial-brand
```

## Bad Names

```text
linear-dashboard
apple-card
tailwind-button
react-modal
purple-glass-panel
```

## Naming by Type

### Components

Components should describe reusable UI primitives.

```text
button
icon-button
input
select
table
drawer
toast
```

### Patterns

Patterns should describe reusable screen sections or page structures.

```text
app-shell
management-list
settings-section
landing-hero
auth-form
```

### Pages

Pages should describe complete user-facing screens.

```text
login
order-management
billing
settings
dashboard
```

### Brand Profiles

Brand profiles should describe style characteristics, not brands.

```text
precise-product-ui
calm-saas
editorial-brand
```
