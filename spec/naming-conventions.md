# 命名规范

AIUI 命名应该稳定、语义化，并且容易被 AI agent 理解。

## 通用规则

- 使用小写
- 使用 hyphen 分隔单词
- 优先使用语义名称，不使用视觉皮肤名称
- 避免框架专属词
- 避免品牌模仿名称

## 推荐命名

```text
management-list
settings-section
filter-toolbar
data-table-workspace
detail-drawer
precise-product-ui
calm-saas
editorial-brand
```

## 不推荐命名

```text
linear-dashboard
apple-card
tailwind-button
react-modal
purple-glass-panel
```

## 按类型命名

### Components

Components 描述可复用 UI primitives。

```text
button
icon-button
input
select
table
drawer
toast
```

### Patterns

Patterns 描述可复用页面区域或页面结构。

```text
app-shell
management-list
settings-section
landing-hero
auth-form
```

### Pages

Pages 描述完整用户界面。

```text
login
order-management
billing
settings
dashboard
```

### Brand Profiles

Brand profiles 描述风格特征，不描述具体品牌。

```text
precise-product-ui
calm-saas
editorial-brand
```
