# AIUI 愿景

## 问题

很多开发者擅长业务逻辑、数据流、系统架构和后端实现，但并不想把大量时间花在前端页面设计上。

AI coding agent 可以很快写出前端代码，但页面经常存在这些问题：

- 看起来像通用 SaaS 模板
- 大量重复卡片
- 随机渐变和装饰
- 信息层级弱
- 间距节奏混乱
- 组件选择不一致
- 页面像 demo，不像真实产品

问题不在于 AI 不会写 CSS，而在于 AI 写 UI 前通常没有清晰的设计意图层。

## 产品定位

AIUI 是给 AI coding agent 使用的结构化 UI 意图规范。

它帮助 AI 理解：

- 页面类型
- 布局结构
- 视觉密度
- 组件组合
- 信息层级
- 品牌方向
- 交互预期
- 需要避免的反模式

AIUI 位于代码生成之前。

```text
用户需求
→ AIUI page spec
→ Agent handoff pack
→ 前端实现
```

## AIUI 优化什么

AIUI 优化的是：

- 更好的前端第一稿
- 更少的设计返工
- 不同 AI agent 生成结构更一致
- 降低开发者对设计能力的依赖
- 让设计知识可以跨项目复用

AIUI 不负责：

- 运行时组件行为
- 后端逻辑
- 完整设计创作
- Figma 像素级还原
- v0.1 的自动视觉评分

## 核心信念

不要让 AI 从空白页开始设计。

AI 应该先获得一份结构化设计说明，这份说明需要：

- 框架无关
- 机器可读
- 人类可审阅
- 必要时项目专属
- 能被不同 AI agent 复用

## 长期目标

AIUI 的长期目标是成为 AI coding 工作流里的通用 UI planning layer。

它不是第一天就宣布自己是标准，而是先证明：同一个需求，使用 AIUI 的 AI agent 可以生成更好的前端页面。
